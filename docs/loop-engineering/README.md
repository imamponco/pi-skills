# Loop Engineering Docs

Contains the orchestration workflow for the full loop system.

## Skill
- `loop-engineering`

## Use when
- you need to coordinate discovery, state, harness, evaluation, git delivery, and scheduling
- you need explicit human checkpoints
- you want the loop to commit, push, and attach work to an existing MR branch when one already exists
- you want branch names, MR titles, commit style, MR descriptions, and local quality checks to stay consistent

## Output
- a loop-engineering plan in Markdown
